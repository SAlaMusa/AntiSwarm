## What It Does

Anti-Swarm detects coordinated inauthentic behaviour on Twitter/X by modelling accounts as nodes in a graph and finding **coordination clusters** — not just individual suspicious accounts. Every flagged account receives a **cluster ID** showing exactly which other accounts it is operating with.

The system routes accounts to one of two detection modes based on posting speed:

| Mode | Coordination signal | Edge weight formula | Trained on |
|---|---|---|---|
| **Mode 1 — Fast swarm** | Device homogeneity | `w = 1 − |H_i − H_j|` | cresci-2017 |
| **Mode 2 — Strategic troll** | Content similarity | `w = cosine_sim(TF-IDF)` | IRA Troll Tweets |

**Key results**

| Evaluation | F1 | AUC | FP Rate |
|---|---|---|---|
| Mode 1 on cresci-2017 | 0.993 | 1.000 | 0.3% |
| Mode 2 hard test (IRA vs bots) | 0.882 | 0.939 | 1.8% |
| Graph vs logistic regression | +14.0 F1 | — | — |
| Kenya Finance Bill 2024 (zero-shot) | — | — | 7.5% flagged, 1 cluster |

```python
import requests

payload = {
    "hashtag": "#RejectFinanceBill2024",
    "context": "Kenya Finance Bill protests — June 2024",
    "platform": "twitter",
    "accounts": [
        {

            "display_name": "@ke_bot_001",  # optional, shown in output
            "avg_delta_time": 4.2,           # mean seconds between tweets
            "delta_time_std": 0.3,           # near-zero = robotic precision
            "device_entropy": 0.04,          # near-zero = single posting client
            "content_diversity": 0.05,
            "account_age_days": 18,
            "follower_following_ratio": 1.0,
            "avg_posts_per_day": 130,
            "is_mobile": 1
        },
        {
            "account_id": "genuine_001",
            "avg_delta_time": 14400,         # ~4 hours between tweets — organic
            "delta_time_std": 9000,          # high variance — human behaviour
            "device_entropy": 0.88,          # many clients — human behaviour
            "content_diversity": 0.78,
            "account_age_days": 1200,
            "follower_following_ratio": 3.8,
            "avg_posts_per_day": 2.1,
            "is_mobile": 1
        }
    ]
}

result = requests.post("http://localhost:8000/score", json=payload).json()

# Top-level verdict
print(f"Risk:     {result['overall_risk']}")
print(f"Flagged:  {result['n_flagged']} / {result['n_accounts']}")
print(f"Clusters: {result['n_clusters']}")
print(f"\n{result['summary']}")

# Cluster details
for cluster in result["clusters"]:
    print(f"\n--- Cluster {cluster['cluster_id']} ---")
    print(f"Members:  {', '.join(cluster['member_ids'])}")
    print(f"Score:    {cluster['mean_score']:.3f}  [{cluster['risk_level']}]")
    print(f"Type:     {cluster['coordination_type']}")
    if cluster.get("delta_time_range"):
        print(f"Timing:   {cluster['delta_time_range']}")
    for signal in cluster["cluster_signals"]:
        print(f"  › {signal}")

# Per-account summary
print("\n--- Accounts ---")
for acc in result["accounts"]:
    flag = "FLAGGED" if acc["is_flagged"] else "clean"
    cid  = f"Cluster {acc['cluster_id']}" if acc["cluster_id"] else "no cluster"
    print(f"  {acc['account_id']:20}  {acc['swarm_probability']:.4f}  "
          f"[{acc['risk_level']:6}]  {cid}")
```

### Computing features from raw tweet data

Use this to extract Anti-Swarm features from a raw tweet CSV before calling the API:

```python
import pandas as pd
import numpy as np
from scipy.stats import entropy as scipy_entropy


def compute_antiswarm_features(tweets_df,
                                uid_col="user_id",
                                ts_col="created_at",
                                source_col="source"):
    """
    Compute Anti-Swarm input features from a raw tweet DataFrame.

    Args:
        tweets_df   : DataFrame, one row per tweet
        uid_col     : column name for account identifier
        ts_col      : column name for tweet timestamp
        source_col  : column name for posting client / app name (optional)

    Returns:
        DataFrame, one row per account, ready for the /score endpoint
    """
    df = tweets_df.copy()
    df[ts_col] = pd.to_datetime(df[ts_col], utc=True, errors="coerce")
    df = df.dropna(subset=[uid_col, ts_col]).sort_values(ts_col)

    records = []
    for uid, g in df.groupby(uid_col, sort=False):
        deltas = g[ts_col].diff().dt.total_seconds().dropna()
        span   = max(1, (g[ts_col].max() - g[ts_col].min()).days)

        # Device entropy: H=0 means single client (bot signature)
        if source_col in g.columns and g[source_col].notna().sum() > 0:
            counts  = g[source_col].value_counts(normalize=True)
            dev_ent = float(
                scipy_entropy(counts) / max(1e-9, np.log(max(2, len(counts))))
            )
        else:
            dev_ent = 0.5  # neutral fallback — no source field

        records.append({
            "account_id":               str(uid),
            "avg_delta_time":           float(deltas.mean()) if len(deltas) > 0 else 3600.0,
            "delta_time_std":           float(deltas.std())  if len(deltas) > 1 else 1800.0,
            "device_entropy":           dev_ent,
            "content_diversity":        0.5,    # set from NLP analysis if available
            "account_age_days":         365.0,  # set from account metadata if available
            "follower_following_ratio": 1.0,    # set from user metadata if available
            "avg_posts_per_day":        len(g) / span,
            "is_mobile":                1
        })

    return pd.DataFrame(records)


# Usage
tweets = pd.read_csv("tweets.csv", low_memory=False)
features = compute_antiswarm_features(tweets)
```

## API Documentation

### `POST /score`

Primary inference endpoint. Accepts account features, returns swarm probabilities, cluster membership IDs, and plain-English coordination signals.

**Request body**

| Field | Type | Required | Description |
|---|---|---|---|
| `hashtag` | string | yes | Hashtag being monitored |
| `context` | string | no | Event description (shown in output) |
| `platform` | string | no | Always `"twitter"` for now |
| `accounts` | array | yes | List of account objects (max 500) |

**Account object fields**

| Field | Type | Default | Description |
|---|---|---|---|
| `account_id` | string | required | Unique account identifier |
| `avg_delta_time` | float | required | Mean seconds between consecutive posts |
| `delta_time_std` | float | required | Std dev of inter-post intervals |
| `device_entropy` | float [0–1] | 0.5 | Normalised Shannon entropy over posting clients |
| `content_diversity` | float [0–1] | 0.5 | Lexical diversity across tweet texts |
| `account_age_days` | float | 365.0 | Days since account creation |
| `follower_following_ratio` | float | 1.0 | Followers divided by following count |
| `avg_posts_per_day` | float | 5.0 | Mean daily tweet volume |
| `is_mobile` | int (0/1) | 1 | Majority of posts from mobile client |
| `display_name` | string | none | Optional label shown in cluster output |

**Response fields**

| Field | Type | Description |
|---|---|---|
| `overall_risk` | string | `HIGH` / `MEDIUM` / `CLEAN` |
| `n_accounts` | int | Total accounts scored |
| `n_flagged` | int | Accounts above threshold |
| `flag_rate` | float | Fraction flagged |
| `n_clusters` | int | Coordination clusters detected |
| `clusters` | array | Cluster objects (see below) |
| `accounts` | array | Per-account results |
| `summary` | string | Plain-English summary |
| `processing_ms` | float | Inference time in milliseconds |

**Cluster object**

| Field | Description |
|---|---|
| `cluster_id` | Integer cluster identifier |
| `size` | Number of member accounts |
| `mean_score` | Mean swarm probability across members |
| `risk_level` | `HIGH` / `MEDIUM` |
| `coordination_type` | `fast-swarm` or `strategic-troll` |
| `detection_mode` | `mode1-timing` or `mode2-content` |
| `member_ids` | List of account IDs in this cluster |
| `cluster_signals` | Plain-English evidence sentences |
| `delta_time_range` | e.g. `"3.8s – 5.1s"` (Mode 1 only) |

## Configuration

Edit `model/routing_config_v09.json` to adjust routing and thresholds without retraining:

```json
{
  "version": "0.9",
  "routing_thresh_seconds": 300,
  "mode1_checkpoint": "antiswarm_mode1_v09.pt",
  "mode2_checkpoint": "antiswarm_mode2_v09.pt",
  "kenya_transfer_thresh": 0.05,
  "kenya_thresh_method": "90th percentile of target distribution scores"
}
```

| Parameter | Default | Effect |
|---|---|---|
| `routing_thresh_seconds` | `300` | Accounts below this threshold (seconds) go to Mode 1 |
| `kenya_transfer_thresh` | `0.05` | Classification threshold for new deployment datasets |
| `mode1_checkpoint` | `antiswarm_mode1_v09.pt` | Path to Mode 1 weights relative to `model/` |
| `mode2_checkpoint` | `antiswarm_mode2_v09.pt` | Path to Mode 2 weights relative to `model/` |

## Troubleshooting

**`FileNotFoundError: model/antiswarm_mode1_v09.pt`**

Model checkpoints are not included in the repository — they are generated by the training notebook. Run `AntiSwarm_GNN_v09_Complete.ipynb` first (GPU runtime recommended), then download the three files from Colab into `model/`.

---

**`ModuleNotFoundError: No module named 'torch_geometric'`**

PyTorch Geometric must match your installed PyTorch version exactly:

```bash
pip install torch-geometric

# If the above fails, install from prebuilt wheels:
TORCH=$(python -c "import torch; print(torch.__version__.split('+')[0])")
pip install pyg_lib torch_scatter torch_sparse \
    -f https://data.pyg.org/whl/torch-${TORCH}+cpu.html
```

**API flags 95%+ of accounts on a new dataset**

The cresci-calibrated threshold does not transfer directly to new deployment populations. Compute the 90th percentile of your dataset's score distribution and use that instead:

```python
import numpy as np, json

result = requests.post("http://localhost:8000/score", json=payload).json()
scores = [acc["swarm_probability"] for acc in result["accounts"]]
transfer_thresh = float(np.percentile(scores, 90))
print(f"Recommended threshold for this dataset: {transfer_thresh:.4f}")

# Persist it
with open("model/routing_config_v09.json") as f:
    cfg = json.load(f)
cfg["kenya_transfer_thresh"] = transfer_thresh
with open("model/routing_config_v09.json", "w") as f:
    json.dump(cfg, f, indent=2)
```

---

**`ValueError: Grouper for 'author_id' not 1-dimensional`**

Your input CSV has two columns that both map to `author_id` after the rename step (e.g. both `Name` and `User ID`). Add this line immediately after loading:

```python
df = pd.read_csv("your_file.csv", low_memory=False)
df = df.loc[:, ~df.columns.duplicated()]   # keep first occurrence of each column name
```

---

**`ValueError: cannot insert user_id, already exists`**

pandas ≥ 2.0 incompatibility with `.groupby().apply().reset_index()`. Use the explicit loop pattern instead:

```python
# Correct — works across all pandas versions
for uid, g in df.groupby(uid_col, sort=False):
    ...

# Broken in pandas >= 2.0 — do not use
df.groupby(uid_col).apply(func).reset_index()
```

---

**Training: `WARNING — bots still slower than genuine accounts`**

This means the timing signal is inverted in your training data. Almost always caused by IRA troll data being included in Mode 1 training. IRA accounts post at ~93,779 second intervals — far slower than genuine users — which reverses the coordination signal. Check that Mode 1 training uses cresci-2017 only and IRA data is separated for Mode 2. The notebook handles this automatically when cells are run in order.

---

**`CUDA out of memory` during training**

Reduce the graph node cap in the notebook:

```python
MAX_NODES = 2000    # reduce from 4000
```

Or switch to a CPU runtime; training will take longer (~3–4 hours) but will complete.

## Responsible Use

Anti-Swarm is an investigative tool for generating hypotheses for human review — not for automated enforcement.

A flag indicates behaviour **consistent with coordination**, not proof of inauthenticity. Collective civic mobilisation can produce posting patterns that superficially resemble bot coordination. All findings should be reviewed by a human investigator before any action is taken, and accounts should not be reported to platforms or named publicly without independent corroboration.

When writing about findings: describe accounts as *"exhibiting coordination signatures consistent with automated posting"* rather than *"bots"* or *"fake accounts"*. Include the cluster evidence — timing ranges, device entropy values, cluster size — as supporting documentation.
